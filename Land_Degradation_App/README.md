# Land Degradation Prediction System

Professional multi-page Streamlit application for environmental AI monitoring of land degradation across Uttar Pradesh, India.

The app presents a green and earth-tone monitoring dashboard with modern metric cards, section dividers, Plotly charts, prediction tools, SHAP explainability, and geographic analysis.

## Quick Start

```bash
cd Land_Degradation_App
pip install -r requirements.txt
streamlit run app.py
```

## Application Pages

| Page | Capability |
|------|------------|
| Home | System Status, monitoring footprint, latest prediction charts, deployed model status, quick navigation |
| Project Overview | Methodology, dataset, feature engineering, and preprocessing summary |
| Single Prediction | Live degradation prediction from environmental inputs and district defaults |
| Batch Prediction | CSV upload, validation, bulk inference, results download, and latest prediction export |
| Model Performance | Baseline comparison, CV scores, evaluation plots, and model diagnostics |
| EDA Dashboard | Interactive filters, KPI cards, distributions, correlations, and scatter analysis |
| SHAP Explainability | Feature importance charts and static SHAP interpretation plots |
| Geographic Dashboard | Choropleth maps, state overview metrics, rankings, trends, district drill-down, and animation |
| Settings | Data update frequency, last dataset update, and manual update trigger |
| About | Technology stack, GeoJSON source, disclaimer, and credits |

## Current UI Features

- Environmental AI monitoring theme using green and earth-tone styling.
- Modern metric cards for study area, predictions, model status, and geographic KPIs.
- Sidebar navigation with monitoring context and operational status.
- Section dividers with plant-themed icons for dashboard readability.
- Plotly charts for latest prediction summaries, EDA, model analysis, and maps.
- Graceful fallback messages when optional latest prediction fields are unavailable.

## Data and Settings

| File | Purpose |
|------|---------|
| `../config/settings.json` | Stores update frequency, last update timestamp, update request timestamp, and pipeline status |
| `../results/latest_predictions.csv` | Latest batch or exported predictions used by the Home System Status charts |
| `../results/predictions.csv` | Full model prediction output from the ML pipeline |
| `../data/ldi_dataset.csv` | Main LDI dataset used by dashboards and district defaults |
| `../models/` | Read-only trained model, preprocessor, and label encoder artefacts |

## System Status Dashboard

The Home page reads:

- `config/settings.json` for Last Data Update and Update Frequency.
- `results/latest_predictions.csv` for Total Predictions and Latest Prediction Year.

It renders these Plotly charts:

- District-wise degradation count.
- Low / Moderate / High percentage pie chart.
- Latest NDVI distribution.

If `results/latest_predictions.csv` is missing, the Home page falls back to `results/predictions.csv` where possible and shows clear messages for unavailable columns.

## Prediction Workflow

Single Prediction uses the trained model artefacts and district median defaults for quick grid-level inference.

Batch Prediction validates uploaded CSV files, runs inference, shows result summaries, and writes the latest output to:

```text
../results/latest_predictions.csv
```

## Important Notes

- Models are read-only and loaded from `../models/`.
- Core datasets are read from `../data/` and are not regenerated by the Streamlit UI.
- Batch Prediction updates `../results/latest_predictions.csv` for the Home dashboard.
- Settings are persisted through `src/config_manager.py` into `../config/settings.json`.
- The Run Update Now button queues or triggers the future update pipeline through `src/update_pipeline.py`.
- GeoJSON boundaries use Datameet Maps, Census 2011 district boundaries, CC BY 4.0.
